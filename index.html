MCPs are now useless

Models are becoming twice as good at coding than every 4 months.

MCps were introduced as a way to introduce tools to these LLM models, however, now a days a lot of tools are included by different providers like executing code or doing websearch

Now that a lot of providers have their own tools, it causes problems when we try to add a lot of tools from our side. Having 10 different tools for example might have the opposite effect that model might not use MCP tools or even use them the wrong way because you're polluting the context with so much MCP information.

A lot of times you have to tell it to use the MCP server to get a certain task done, which is not the behavior we want from an agentic model.

There are really only 3 MCP servers that will help us in development:

  • Read & Validate Data
  • Use the browser
  • Get up-to-date documentation

Cursor and a lot of all these other AI tools already have a lot of these capabilities without the need of polluting the context.

They can use the command line to interact with git, which allows them to interact with sqlite for databases.
